Jamming

Playlist Creator React.js App using Spotify's API

Jamming is a React App that allows users to search the Spotify library, create a custom playlist, then save it to their Spotify account. Like my Ravenous project, I broke the project down into four parts of production, first, creating the static components, second, passing information to those components, third, setting the state of the components and lastly, interacting with the Spotify API.

I broke the components down into App, SearchBar, SearchResults, PlayList, TrackList and Track, building out the CSS files for each, writing the HTML inside the render methods, importing and exporting the necessary components, and adding the methods for Track Search, renaming a PlayList, saving a PlayList to Spotify, removing a Track and adding a Track to a PlayList. After this, I passed the state of a search results parameter through a series of components to render an array of tracks. Next I passed the state of a userâ€™s custom playlist title and tracks from the App component down to components that render them. I then added methods for adding and removing songs from the playlist, then methods for updating the playlist's title and adding and removing a song from the search results to the user's custom playlist. After this, I set up the SearchResults section that allowd the search query to be sent to the Spotify API and display the return data. The Spotify.js component contains the API link that allows this to happen. I created this by registering with Spotify and recieving an API access token and using it to link the GET and POST requests I have in other files to the Spotify API. Once this was done I was ready to create the savePlaylist method that sent the playlist and track information back to Spotify to save.

I hop on this project every once in a while to add small features here and there when I think of new ones. I hope you get the chance to look around and let me know if you can think of anything I should add.
