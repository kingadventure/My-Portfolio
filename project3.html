A PHP/MySQL App for submitting and retrieving contacts

This Fake Contact Form is meant to show the simple functionality of inserting and storing data in a database then querying by selected filter for retrieval. I broke this project down into 4 parts, building the basic form using HTML and Web Responsive CSS, Creating the MySQL database and table that the site would use, linking the database and writing the PHP and MySQL insertion code for the submission form on the right, then writing the PHP and query code by the filters to return the results.

After building the basic HTML page, I used PhpMyAdmin to build out the database and within the database the table that would hold all the contact data. Once the permissions was set up, I added a PHP file to my public_html folder that would serve as the link to the database for any other PHP file that needed it. Next was coding the PHP for the right-sided submission form. It was constructed accordingly, including the link to the database, running all the form fields through a mysqli_real_escape_string() function to insure security, if statements for all of the fields to insure the form was fully filled out, creating the constant for the SQL insertion with the filled-in values, then the query function fulfilling the insertion. That was it for the right form. For the left form I chose to imbed the PHP in the index page rather than creating a seperate file. The PHP included the link to the database, the mysqli_real_escape_string() function again, then if statements to check what filter was used. Within each if statement was the necessary SQL code to search the sumbit term in the filter's column, the if statement that returned true gave the SQL code to pull the data. After this I wrote code to format the results for easy readability and that was it!

I don't have any form of real security for this app, hence the "fake" contact database versus a real one. However, because of this there have been a lot of funny submissions. I hope you get the chance look around and enjoy!
